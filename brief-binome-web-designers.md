# Brief binome web designers - Andenne Bears

Date : 24 avril 2026  
Base de travail :

- `refonte-contenu-andenne-bears.md`
- `designer-ux-architecture.md`
- `designer-ui-direction-visuelle.md`

## 1. Decision commune

La refonte doit transformer le site actuel, qui fonctionne surtout comme une carte de visite, en site vitrine utile pour recruter, rassurer, informer et ouvrir la porte aux partenaires.

Priorite commune : le parcours `Venir essayer`.

Le site doit repondre rapidement a quatre questions :

- Qui sont les Andenne Bears ?
- Quels sports peut-on pratiquer : football americain tackle et flag football ?
- Est-ce accessible a mon profil ou a mon enfant ?
- Comment venir essayer ou contacter le club ?

## 2. Direction UX retenue

Navigation principale :

- Accueil
- Rejoindre
- Equipes
- Calendrier
- Partenaires
- Contact

Parcours prioritaires :

- Joueur debutant : comprendre, choisir une discipline, demander un essai.
- Parent : verifier age, securite, encadrement, horaires, lieu et prix.
- Sponsor : comprendre la valeur locale et contacter le comite.
- Supporter : trouver calendrier, evenements et reseaux sociaux.

CTA principal global :

- `Venir essayer`

CTA secondaires :

- `Contacter le club`
- `Voir les entrainements`
- `Poser une question parent`
- `Devenir partenaire`
- `Voir le calendrier`
- `Suivre le club`

## 3. Direction visuelle retenue

Le site doit etre sportif, local, humain et accessible. Il doit montrer le club, pas seulement le logo.

Principes visuels :

- Utiliser une vraie photo d'equipe, d'entrainement ou de match dans le hero.
- Garder une interface sombre et contrastee avec l'or comme couleur d'action.
- Rester lisible pour les parents, debutants et partenaires.
- Eviter une ambiance trop agressive ou trop corporate.

Palette recommandee :

- Noir Bears : `#080808`
- Anthracite terrain : `#171717`
- Blanc chaud : `#F7F4EC`
- Or Bears : `#F2CB2A`
- Or fonce : `#B88919`
- Rouge brun logo : `#7A2418`, en accent rare

Typographies :

- Titres : `BertholdCityBold` ou `BertholdCityMedium` si les fichiers sont conserves.
- Texte courant : police systeme ou `Inter`/`Roboto`.
- CTA : courts, lisibles, graisse forte.

## 4. Structure de page recommandee

### Hero

Objectif : comprehension immediate.

Contenu :

- Badge : `Football americain et flag football a Andenne`
- H1 : `Andenne Bears`
- Texte : `Depuis 2002, les Bears rassemblent joueurs, joueuses, coachs et benevoles autour d'un meme esprit : One team. One family. One heartbeat.`
- CTA principal : `Venir essayer`
- CTA secondaire : `Contacter le club`

Design :

- Photo reelle en fond ou pleine largeur.
- Overlay sombre solide.
- Logo present, mais pas comme seul element identitaire.
- Sur mobile, CTA visible dans le premier ecran.

### Rejoindre

Objectif : convertir les curieux.

Contenu :

- Debutants bienvenus.
- Etapes simples : contacter, recevoir confirmation, venir tester.
- Choix ou orientation : tackle / flag / je ne sais pas encore.
- Formulaire court.

### Disciplines

Objectif : expliquer sans jargon.

Contenu :

- Football americain tackle.
- Flag football sans contact.
- Ages, categories et horaires uniquement apres validation.

Design :

- Deux cards fortes, avec photo ou pictogramme.
- Badges utiles : `Debutants bienvenus`, `A confirmer`, `Mixte`, `Sans contact`.

### Infos pratiques

Objectif : eviter que les visiteurs cherchent sur Facebook.

Contenu :

- Horaires.
- Lieux.
- Age minimum.
- Materiel pour essai.
- Cotisations si validees.
- Derniere mise a jour.

Point critique :

- Les sources publiques divergent. Afficher seulement les donnees confirmees par le comite.

### Vie du club

Objectif : donner envie.

Contenu :

- Photos reelles.
- Citation courte joueur, coach, parent ou benevole.
- Valeurs : combativite, respect, solidarite, ambiance familiale.

### Calendrier

Objectif : donner du rythme.

Contenu :

- Prochains matchs.
- Entrainements ouverts.
- Evenements.
- Etat vide utile si aucun evenement n'est renseigne.

### Partenaires

Objectif : ouvrir le sponsoring.

Contenu :

- Pourquoi soutenir les Bears.
- Visibilite locale.
- Sport amateur et jeunesse.
- Logos partenaires.
- CTA : `Devenir partenaire`.

### Contact

Objectif : action finale.

Contenu :

- `comite@andenne-bears.be`
- Facebook
- Instagram
- Formulaire essai/contact.
- Adresse uniquement apres validation.

## 5. Points a arbitrer avant maquette finale

- Adresse actuelle du terrain.
- Adresse habituelle du terrain.
- Existence et duree d'un lieu temporaire a Evelette-Jallet / Ohey.
- Horaires tackle.
- Horaires flag football.
- Categories actives : U13, U16, seniors, autres.
- Ages minimums.
- Cotisations exactes.
- Nom officiel : `Andenne Bears`, `Namur Country Andenne Bears` ou `Namur County Andenne Bears`.
- Photos disponibles et droits d'utilisation.
- Contacts publics par role : recrutement, parents, sponsors.

## 6. Repartition du travail

Designer UX :

- Finaliser l'arborescence.
- Prioriser les parcours.
- Produire wireframes mobile et desktop.
- Definir le contenu fonctionnel de chaque section.
- Specifier les etats vides et les etats `information a confirmer`.

Designer UI :

- Produire la direction visuelle haute fidelite.
- Definir palette, typographies, composants et usage photo.
- Maquetter le hero, les cards, les formulaires, les sections infos pratiques et partenaires.
- Verifier lisibilite, contraste et mobile.

Travail commun :

- Valider que le CTA `Venir essayer` reste prioritaire.
- Aligner l'ordre des sections.
- Tester la page sur mobile.
- Lister les contenus impossibles a finaliser sans retour du club.
- Eviter une dependance excessive aux actualites frequentes.

## 7. Prochaine etape recommandee

Avant design final, organiser un retour club de 30 minutes avec cette checklist :

- Valider horaires, lieux, categories, prix et nom officiel.
- Choisir 10 a 20 photos exploitables.
- Confirmer le contact public.
- Decider si le lancement se fait en page unique ou en mini-site multi-pages.
- Confirmer les priorites : recrutement, parents, partenaires, calendrier.

Apres validation, les designers peuvent produire :

- Wireframe mobile.
- Wireframe desktop.
- Maquette haute fidelite mobile.
- Maquette haute fidelite desktop.
- Mini design system.
- Liste des contenus manquants pour developpement.
